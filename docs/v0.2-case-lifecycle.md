# TRACE v0.2 case lifecycle

Milestone 8.2 introduces operational metadata and controlled lifecycle transitions for persisted investigations.

## Case metadata

Each case can retain:

- priority: `low`, `normal`, `high`, or `critical`;
- a public-safe external reference;
- a concise redacted investigation summary.

Metadata is stored inside the existing investigation snapshot. This keeps the change backward-compatible and avoids an unnecessary database migration. Older snapshots deserialize with `normal` priority and empty optional metadata.

## Controlled transitions

Normal lifecycle transitions are explicit:

- `draft` -> `evidence_validated`;
- `evidence_validated` -> `draft` or `analyzed`;
- `analyzed` -> `reviewed`;
- `reviewed` -> `analyzed` or `exported`;
- `exported` -> `reviewed`.

Archive and reopen remain dedicated operations. Archived cases cannot be edited or analyzed until reopened.

Analysis routes preserve the persisted case identity and metadata while replacing the current evidence snapshot and appending a new immutable analysis run.
