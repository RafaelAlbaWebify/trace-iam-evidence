# TRACE v0.2 — Investigation Workbench roadmap

## Product standard

TRACE v0.2 is not a demonstration layer over isolated analysis forms. It is a real, local-first support-engineering investigation workbench. Every capability must operate on persisted cases, preserve evidence provenance and immutable history, enforce safety boundaries in the backend, and pass automatic browser and cross-platform proof before merge.

## Milestone 8 — Real case management

### 8.1 Persisted investigation workspace — complete

Deliverables:

- operator-created investigations with server-generated IDs;
- persisted draft state before the first analysis;
- case title and scenario selection;
- active-case workspace and visible case identity;
- existing cases reopened from the case register;
- scenario-specific analysis locked to the active case;
- archive/reopen protection;
- immutable runs and exports attached to the generated case ID;
- server-side rejection of scenario, title, and archive-state mismatches.

Exit proof:

- all three browser workflows create real persisted cases;
- generated IDs match the documented `trace-<12 hex characters>` contract;
- analysis, history and exports use the generated ID rather than fixed demonstration IDs;
- Conditional Access browser proof archives and reopens the same case;
- backend and frontend checks pass on Ubuntu and Windows;
- release publisher and Ubuntu/Windows release packs remain green.

### 8.2 Investigation lifecycle and case metadata — in progress

Implemented:

- priority classification: low, normal, high and critical;
- public-safe external case reference;
- concise redacted case summary;
- metadata editing without changing case identity or immutable runs;
- backward-compatible loading of older persisted snapshots;
- explicit lifecycle transition contract;
- backend rejection of invalid lifecycle jumps;
- metadata preservation through all three analysis routes;
- archive protection for edits and analysis.

Remaining exit proof:

- automatic API tests pass on Ubuntu and Windows;
- existing persistence and browser workflows remain green;
- metadata is exposed clearly in the operator UI;
- immutable lifecycle events are added under Milestone 11 rather than simulated prematurely.

## Milestone 9 — Evidence workspace

- multiple evidence items per case;
- structured forms, CSV, JSON, log excerpts and operator notes;
- evidence source, collection time, reliability, redaction and notes;
- evidence preview and validation before analysis;
- exact immutable evidence snapshot attached to each run.

## Milestone 10 — Structured findings workspace

- finding cards with severity, confidence and rule version;
- supporting, contradicting and missing evidence views;
- separate safe checks, non-actions, limitations and unresolved questions;
- filtering and export while preserving raw JSON and Markdown.

## Milestone 11 — Investigation timeline

- system events for case creation, evidence changes, analyses, status changes and exports;
- redacted operator notes;
- immutable generated events and clearly attributed manual notes;
- chronological and newest-first views.

## Milestone 12 — Run comparison

- compare any two runs in a case;
- added, resolved and changed findings;
- confidence, evidence and ruleset differences;
- explicit explanation of whether changes came from evidence or rule versions;
- comparison report export.

## Milestone 13 — Search and operational dashboard

- search by title, redacted case reference and generated ID;
- filters for status, scenario, findings and date;
- operational workload cards without vanity metrics;
- recently updated and waiting investigations.

## Milestone 14 — Local product packaging

- one-command Windows start, stop and diagnostics workflow;
- prerequisite checks, migrations and service startup;
- configurable local data directory;
- SQLite backup and restore;
- portable review ZIP generated directly in Downloads without automatically opening the folder.

## Milestone 15 — Verified v0.2.0 release

- complete case-to-evidence-to-analysis-to-comparison browser workflow;
- multiple immutable runs and timeline events;
- search, reopen, export, backup and restore proof;
- Ubuntu and Windows CI and release packs;
- controlled Windows review;
- stable `v0.2.0` publication from the verified merge commit.

## Deferred beyond v0.2

Live Microsoft Graph connectivity, credential storage, automated remediation, direct access or policy changes, multi-user cloud hosting and AI-generated root-cause claims remain out of scope until the local investigation model and security design are mature and separately approved.
