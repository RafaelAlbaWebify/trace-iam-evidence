# TRACE IAM Evidence v0.3.0

TRACE v0.3.0 is the completed operational-interface and portfolio release of the local-first, read-only IAM evidence investigation workbench.

## Highlights

- Persistent responsive application shell and operational dashboard.
- Persisted case register, evidence inventory and lifecycle controls.
- Conditional Access, resource assignment and Guest/B2B lifecycle workflows.
- Structured findings with supporting, contradicting and missing evidence.
- Explicit safe checks, non-actions, limitations and uncertainty.
- Immutable analysis runs, append-only timeline and redacted operator notes.
- Run comparison, investigation history and Markdown/JSON report access.
- Self-locating Windows runtime management, verified SQLite backup/restore and portable public-safe review ZIP.

## Verification

The exact release line is validated through:

- backend lint, strict typing and tests on Ubuntu and Windows;
- frontend type checking, tests and production builds on Ubuntu and Windows;
- Chromium acceptance for all three supported scenarios;
- Windows runtime lifecycle proof;
- portable-review safety, integrity and repeatability proof;
- Ubuntu and Windows release-proof packs;
- desktop and narrow GUI screenshots plus browser diagnostics.

The final GUI proof recorded zero console errors, zero page errors, zero duplicate IDs, zero unnamed enabled buttons and no document-level horizontal overflow at 390 px.

## Safety boundary

TRACE remains local-first and read-only. It accepts only redacted or public-safe evidence and does not connect to Microsoft Graph, scan a tenant, grant access, change invitations, modify policy, weaken controls or automate remediation.

## Repository history

This repository is the canonical maintained TRACE project. The earlier `trace-ops` repository is archived and preserved only as development history.

## Upgrade notes

v0.3.0 does not introduce a new backend or persistence contract over v0.2.0. The release primarily completes the operational interface, public presentation and browser-proof coverage while preserving existing APIs, data semantics, immutable runs and safety boundaries.
